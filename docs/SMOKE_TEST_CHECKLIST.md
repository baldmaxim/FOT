# Smoke Test Checklist

Ручной чеклист перед эксплуатационной приёмкой. 10 сценариев, покрывающих критические пути.

---

## 1. Login + 2FA

**Шаги:**
1. Открыть `/login`
2. Ввести корректные email/пароль
3. Если включён 2FA — ввести TOTP-код
4. Убедиться, что произошёл редирект на главную страницу роли

**Ожидаемый результат:** JWT получен, сайдбар отображает имя и роль пользователя.

**Провал:** Белый экран, ошибка 401/500, редирект обратно на `/login`.

---

## 2. Profile load

**Шаги:**
1. Войти под любой ролью
2. Проверить сайдбар: ФИО, роль, организация
3. Для `worker` — открыть `/employee`, проверить карточку профиля

**Ожидаемый результат:** ФИО расшифровано и отображается корректно, роль соответствует `position_type`.

**Провал:** ФИО показывает зашифрованный текст или `null`, роль пустая.

---

## 3. Role access

**Шаги:**
1. Войти как `worker` — попробовать открыть `/admin/users`
2. Войти как `header` — попробовать открыть `/skud-settings`
3. Войти как `admin` — проверить доступ ко всем разделам

**Ожидаемый результат:** `worker` и `header` получают редирект или 403. `admin` видит все страницы.

**Провал:** `worker` видит админ-панель, `admin` не может открыть раздел.

---

## 4. Employees paginated list

**Шаги:**
1. Открыть список сотрудников (GET `/api/employees?page=1&limit=20`)
2. Проверить, что отображается первая страница
3. Перейти на страницу 2
4. Ввести поисковый запрос — проверить фильтрацию

**Ожидаемый результат:** Список загружается, пагинация работает, поиск фильтрует по ФИО.

**Провал:** Пустой список при наличии данных, пагинация не переключается, поиск возвращает всех.

---

## 5. Employee detail

**Шаги:**
1. Кликнуть на сотрудника в списке
2. Проверить карточку: ФИО, должность, отдел, организация
3. Открыть вкладку «История» (если есть)

**Ожидаемый результат:** Все поля заполнены, история отображает записи с датами.

**Провал:** 404 при открытии, пустые поля, ошибка расшифровки ФИО.

---

## 6. Employee SKUD events

**Шаги:**
1. Открыть карточку сотрудника с привязкой к Sigur
2. Перейти в раздел СКУД-событий
3. Выбрать период (последние 7 дней)
4. Проверить таблицу событий: дата, время, точка доступа, направление

**Ожидаемый результат:** Таблица заполнена, события отсортированы по дате/времени.

**Провал:** Пустая таблица при наличии данных в БД, ошибка загрузки, некорректные даты.

---

## 7. Structure page

**Шаги:**
1. Войти как `header`+ и открыть `/admin/structure`
2. Проверить дерево организаций и отделов
3. Раскрыть вложенные отделы

**Ожидаемый результат:** Дерево загружается, иерархия отделов корректна, вложенность отображается.

**Провал:** Пустая страница, плоский список вместо дерева, отделы без родителей.

---

## 8. Audit logs

**Шаги:**
1. Войти как `admin`
2. Открыть `/admin/audit`
3. Запустить полный аудит (GET `/api/audit/run`)
4. Проверить результаты проверок

**Ожидаемый результат:** Аудит завершается, отображаются результаты по каждой проверке.

**Провал:** Таймаут, 500 ошибка, пустой ответ.

---

## 9. Search unlinked employee (admin)

**Шаги:**
1. Войти как `admin`+
2. Открыть интерфейс привязки сотрудников (поиск в админке)
3. Ввести часть ФИО — проверить результаты поиска
4. Убедиться, что отображаются только непривязанные сотрудники

**Ожидаемый результат:** Поиск возвращает релевантные результаты, фильтрация по статусу привязки работает.

**Провал:** Поиск возвращает пустой список, показывает уже привязанных, ошибка 500.

---

## 10. Sigur sync + import summary

**Шаги:**
1. Войти как `admin`, открыть `/skud-settings`
2. Запустить синхронизацию событий за 1 день
3. Дождаться завершения, проверить summary:
   - `sigurTotal` — общее кол-во событий из Sigur
   - `imported` — импортировано
   - `skipped` — пропущено (дубликаты)
   - `matched` — сопоставлено с сотрудниками
   - `errors` — массив ошибок (должен быть пуст)
4. Повторный запуск за тот же день — `imported` должен быть 0, `skipped` > 0

**Ожидаемый результат:** Summary отображается полностью, повторный импорт корректно дедуплицируется.

**Провал:** Summary не появляется, `errors` не пуст, повторный импорт дублирует записи.

---

## Топ-3 критичных сценария

| # | Сценарий | Почему критичен |
|---|----------|-----------------|
| 1 | Login + 2FA | Без входа система недоступна |
| 2 | Employees paginated list | Основной рабочий экран для всех ролей |
| 3 | Sigur sync + import summary | Ядро интеграции со СКУД, потеря данных = потеря учёта |
