# scripts/archive — отработавшие одноразовые миграции

Здесь хранятся скрипты, которые были однократно выполнены на проде и больше не нужны.
Их оставляем в репо ради git-истории, но из активного `scripts/` убираем,
чтобы не путать с переиспользуемыми утилитами (`audit-route-protection`,
`setup-s3-cors`, `migrate-patent-receipts-to-encrypted`, `create-test-code`).

| Скрипт | Дата применения | Что делал |
|--------|------------------|-----------|
| `backfill-dedup-hash.ts` | апрель 2026 | Заполнял dedup-hash для существующих `skud_events`, дедуплицировал записи |
| `backfill-employee-ids.ts` | апрель 2026 | Заполнял `employee_id` в дочерних таблицах после нормализации схемы |
| `freeze-transfer-history.ts` | 30 апреля 2026 | Снимок истории переводов сотрудников на дату заморозки |
| `import-manager-department-access.ts` | апрель 2026 | Импорт иерархии «начальник → отдел» из файла |
| `push-employee-1010-name-to-sigur.ts` | 29 апреля 2026 | Точечный апдейт ФИО сотрудника id=1010 в Sigur (разовая правка) |
| `recalc-daily-summaries.ts` | апрель 2026 | Пересчёт `skud_daily_summary` после bug-fix таймзоны |
| `restore-mass-fired.ts` | 30 апреля 2026 | Реактивация ошибочно уволенных (запуск с флагом `--apply`) |

Перед повторным запуском любого из них — внимательно прочитать заголовок файла
и проверить, что условие применимости всё ещё валидно.
Большинство либо идемпотентны через флаг `--apply`/dry-run, либо предполагают
конкретное состояние БД на момент применения.
